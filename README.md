# 學代會歷年預算審查系統

## Proposal Report

### 動機與目標
學生代表在審查預算時，常需要閱讀大量表格資料，並在有限時間內理解各部門預算內容、比較不同項目的差異，並從複雜的科目表中挑出可能不合理的編列。這樣的審查工作若完全依賴人工逐筆比對，不但耗時，也容易因資料量過大而忽略重要問題。

本人與多位盟友將於下學期擔任學生代表。我們之所以參選，正是因為認為學生會部分預算中存在人事費用過度膨脹、執行率偏低等問題，但過去審查時，對此現象的系統性檢視相對不足。若能開發一套系統，自動整理歷年預算與決算資料，並依照特定審查規則標記人事費過度膨脹的項目，將有助於降低審查負擔，也能讓審查更聚焦於真正需要說明或進一步處理的部分。而在大部份情況下，由於行政部門怠政，往往需要凍結才願意繳交詳細工作狀況，這樣的自動凍結應能有效讓學代以最低成本要求學生會認真面對預算審查。

因此，本專題希望開發一套「學代會歷年預算審查支援系統」，將歷年預算表與決算資料轉換為可查詢、可比較的資料系統，協助使用者更有效率地閱讀與分析預算資料，並提供規則式審查輔助功能，快速找出值得注意的預算項目。

以下為本系統目前預計優先支援的核心審查規則：

1. **工作費、臨時工資、稿費、製圖費合計不得大於上期決算執行額之 120%，且不得大於該部門非會費收入之和。超出之部分，列為建議凍結範圍**  
   這四種費用可視為學生會人事相關支出。若其總額較上期決算增加超過 20%，卻沒有對應增加的自有財源，則可能存在過度膨脹的問題，應列為優先審查項目。

2. **某部門人事相關費用的成長率，不得高於該部門整體預算成長率的 110%。超出之部分，列為建議凍結範圍**  
   有時問題不在於單一科目是否超過門檻，而在於整體預算並未明顯增加，但人事相關費用卻大幅上升。此種情況可能顯示部門支出結構失衡，因此也應納入審查重點。

3. **若某人事費項目上期決算執行率低於 80%，且本期預算數高於上期決算執行數，則超出上期決算執行數之部分，列為建議凍結範圍。**  
   若某項目上期實際執行率偏低，代表其原先編列可能已高於實際需求。若本期預算仍高於上期實際執行數，則可能存在浮編或需求評估過高的情況，應列為優先審查項目。

若預算項目超出上述規則，系統將標記其為需優先檢視項目，並試算超出部分的建議凍結金額，供學生代表審查時參考。

本專題的目標為透過資料整理、跨年度比較與規則判定，提供一套可協助學代進行預算初步審查的工具。

### 競品比較
在網路中與本專題完全相近的系統相對較少，目前較接近的實務作法是利用 Excel 進行手動審查與公式比對。

1. **Excel 手動審查**  
   這是目前最可能被實際使用的方法。審查者可自行整理歷年預算與決算資料，利用公式計算總額、比例、增減幅度，並手動找出異常項目。其優點是工具普及、上手門檻低、彈性高；但缺點在於資料整理成本高、公式容易出錯、不同年度格式不一致時處理困難，且當審查規則數量增加時，維護與檢查成本也會迅速上升。此外，Excel 雖然能計算數值，但較難提供一致的審查流程、重點標記與系統化的優先審查清單，因此本專題希望在此基礎上提供更有效率的審查支援工具。

### 預期功能
1. **歷年預算與決算資料匯入**
   - 匯入歷年 Excel 預算表與決算資料
   - 將不同年度資料整理成統一格式

2. **預算架構瀏覽**
   - 顯示預算的階層結構，例如款、項、目、節
   - 支援逐層展開與收合，方便閱讀各部門與各科目內容

3. **搜尋與查詢**
   - 可依科目名稱、部門、年度或關鍵字搜尋預算項目
   - 快速定位特定支出或收入項目

4. **跨年度比較**
   - 比較同一部門、同一科目或同一業務在不同年度的預算與決算變化
   - 顯示增減金額與變動幅度

5. **規則式審查判定**
   - 依照預先設定的審查規則，自動標記可能不合理的預算項目
   - 顯示超出金額與觸發原因

6. **凍結建議功能**
   - 對於超出審查規則的部分，自動計算建議凍結的金額
   - 提供學代作為審查時的初步參考

7. **審查標記與備註**
   - 可將項目標記為「待審查」、「需說明」、「建議凍結」等狀態
   - 可新增簡短備註，方便後續整理審查意見

8. **優先審查清單**
   - 自動整理所有被規則觸發的預算項目
   - 形成重點審查清單，協助使用者快速掌握高風險項目

### 使用技術
- **Python**：主要開發語言
- **pandas**：處理與整理預算、決算資料
- **openpyxl**：讀取 Excel 檔案
- **Streamlit**：建立簡易操作介面
- **Git / GitHub**：版本管理與作業繳交

若後續有額外時間，可能再補充資料視覺化或匯出功能。

### Prototype 預計可驗證內容
在 Prototype 階段，我預計先完成系統的核心資料處理與基本審查功能，並驗證以下內容：

1. **可成功匯入至少數個年度的預算表與決算資料**
   - 驗證系統能正確讀取 Excel 檔案
   - 驗證不同年度資料可整理成一致格式

2. **可完成基本的預算架構瀏覽**
   - 驗證系統能以階層方式顯示部門與科目資料
   - 驗證使用者能查看特定年度與特定部門的預算內容

3. **可進行基本搜尋與跨年度比較**
   - 驗證使用者能依關鍵字或部門查找預算項目
   - 驗證系統可比較同一項目在不同年度的變化

4. **可驗證至少一到兩項審查規則**
   - 驗證系統能依規則自動標記可能不合理的預算項目
   - 驗證系統能顯示觸發規則的原因與超出範圍

5. **可初步產生凍結建議或重點審查清單**
   - 驗證系統能整理出優先審查的項目
   - 驗證系統可作為學代審查時的初步輔助工具

Prototype 階段的重點將放在資料結構整理、跨年度比較與規則式判定是否可行，介面則以簡單清楚、可操作為主。
