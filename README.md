# [學代會歷年預算審查支援系統]

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
學生代表在審查預算時，常需要閱讀大量表格資料，並在有限時間內理解各部門預算內容、比較不同項目的差異，並且需要在複雜的科目表中挑出不合理的預算。在預算審核中，往往需要各部門跨年進行比較，才能避免預算過度膨脹。然而，預算書通常以 Excel 或表格形式呈現，雖然資料完整，但在搜尋、比較、整理與標記上較不方便，容易增加審查負擔。此外，因為現有預算表是分學期編列，如果想要跨年份比較與整理，需逐一下載歷年資料自行比對，不僅不直觀且徒增困擾。因此如果有一個系統能夠完成上述任務，應該能有效提升審核效率，也能更嚴格檢視預算支用的合理性。

本專題希望開發一套「學代會歷年預算審查支援系統」，將歷年預算表轉換為可查詢、可比較的資料系統，協助使用者更有效率地閱讀與分析預算資料。提供審查輔助工具，幫助使用者快速找出值得注意的預算項目。

### 預期功能
<!-- 列出預計實作的功能 -->
1. **預算資料匯入**
   - 匯入歷年 Excel 預算表
   - 將不同年度資料整理為統一格式

2. **預算架構瀏覽**
   - 顯示預算的階層結構，例如款、項、目、節
   - 支援逐層展開與收合

3. **搜尋與查詢**
   - 可依名稱、代碼、部門或關鍵字搜尋預算項目

4. **排序與篩選**
   - 可依金額、占比等欄位排序
   - 可篩選特定年度、部門或類型的預算項目

5. **跨年度比較**
   - 比較同一項目或同一部門在不同年度的預算變化
   - 顯示增減金額與變動情形

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
本專題雖然主題是預算審查，但實作過程中會用到資料結構與進階程式設計的概念。

1. **樹狀結構（Tree）**  
   預算表中的款、項、目、節具有明確的階層關係，因此可將預算資料表示為樹狀結構，方便逐層瀏覽與整理。

2. **雜湊表（Hash Map / Dictionary）**  
   系統需要依照代碼、名稱、年度或部門快速查詢資料，因此可使用 dictionary 建立索引，提高搜尋效率。

3. **排序與篩選（Sorting / Filtering）**  
   預算審查需要快速找出高額、占比偏高或變動較大的項目，因此排序與篩選是本系統的重要功能。

4. **資料解析與轉換（Parsing）**  
   原始資料來自 Excel 預算表，需要先清理並轉換為程式可處理的結構化資料，這涉及資料表示與前處理設計。

5. **狀態管理（State Management）**  
   若系統支援標記、備註與審查進度追蹤，便需要管理每個預算項目的狀態，例如是否已檢視、是否列入關注清單等。

6. **模組化設計**  
   本專題可分成資料匯入、資料處理、查詢比較、標記管理與介面顯示等模組，與進階程式設計中強調的程式結構化與模組化設計有關。
