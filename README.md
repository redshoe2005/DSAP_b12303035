# 學代會歷年預算審查系統

## Proposal Report

### 動機與目標

學生代表在審查預算時，常需要閱讀大量表格資料，並在有限時間內理解各部門預算內容、比較不同項目的差異，並從複雜的科目表中挑出可能不合理的編列。這樣的審查工作若完全依賴人工逐筆比對，不但耗時，也容易因資料量過大而忽略重要問題。

本人與多位盟友將於下學期擔任學生代表。我們之所以參選，正是因為認為學生會部分預算中存在人事費用過度膨脹、執行率偏低等問題，但過去審查時，對此現象的系統性檢視相對不足。若能開發一套系統，自動整理歷年預算與決算資料，並依照特定審查規則標記人事費過度膨脹或執行率偏低的項目，將有助於降低審查負擔，也能讓審查更聚焦於真正需要說明或進一步處理的部分。

因此，本專題希望開發一套「學代會歷年預算審查支援系統」，將歷年預算表與決算資料轉換為可查詢、可比較的資料系統，協助使用者更有效率地閱讀與分析預算資料，並提供規則式審查輔助功能，快速找出值得注意的預算項目。

本系統目前將「工作費、臨時工資、稿費、製圖費」視為人事相關費用，並以「審查單位／小組」作為主要分析粒度。一般單位以部門底下的單位為準；但對於學術部與文化部，若其底下另有小組，則會將各小組獨立列入比較，例如「學術部｜永續組」。此外，司法部門與選舉罷免執行委員會不納入自動審查規則，以避免將具有特殊性質的機構誤列為一般行政單位。

目前 prototype 採用兩項核心審查規則：

1. **人事相關費用合理性審查**  
   工作費、臨時工資、稿費、製圖費合計，原則上不得超過合理人事費上限。系統會同時考量歷史決算水準、整體預算成長幅度，以及本期可辨認的非會費收入，計算建議凍結金額。

   ```text
   上限 A = 上期人事相關決算 × 120%
   上限 B = 上期人事相關決算 × (1 + 整體預算成長率 × 110%) + 本期非會費收入
   合理人事費上限 = max(上限 A, 上限 B)
   建議凍結金額 = max(0, 本期人事相關預算 - 合理人事費上限)
   ```

   此規則的目的不是單純懲罰人事費增加，而是判斷人事費增加是否有合理基礎。若某單位的人事費雖然成長，但整體預算也同步增加，或該單位能找到專案補助、活動收入等非會費財源，系統會將這些因素納入考量。只有當人事費明顯超過歷史水準，且缺乏足夠財源或業務規模支持時，才會列為建議凍結範圍。

2. **低執行率人事科目凍結建議**  
   若某審查單位或小組的人事相關科目，上期決算執行率低於 80%，且本期同一科目的預算合計高於上期決算執行數，則超出上期決算執行數之部分，列為建議凍結範圍。

   ```text
   比較粒度 = 審查單位／小組 × 科目

   若 上期同單位同人事科目決算合計 / 上期同單位同人事科目預算合計 < 80%
   且 本期同單位同人事科目預算合計 > 上期同單位同人事科目決算合計
   則 建議凍結金額 = 本期同單位同人事科目預算合計 - 上期同單位同人事科目決算合計
   ```

此規則的設計原因在於，每一年度的計畫名稱可能不同，但單位底下的會計科目通常具有延續性。因此，系統不以「計畫」作為比對單位，而是將同一審查單位或小組底下的同一人事相關科目加總後再進行比較。這可以避免因年度計畫名稱改變而導致比對失準。

### 競品比較

在網路中與本專題完全相近的系統相對較少，目前較接近的實務作法是利用 Excel 進行手動審查與公式比對。

1. **Excel 手動審查**  
   這是目前最可能被實際使用的方法。審查者可自行整理歷年預算與決算資料，利用公式計算總額、比例、增減幅度，並手動找出異常項目。

   其優點是工具普及、上手門檻低、彈性高；但缺點在於資料整理成本高、公式容易出錯、不同年度格式不一致時處理困難，且當審查規則數量增加時，維護與檢查成本也會迅速上升。此外，Excel 雖然能計算數值，但較難提供一致的審查流程、重點標記與系統化的優先審查清單。

2. **一般資料視覺化工具**  
   例如 Google Sheets、Power BI 或 Tableau 等工具，可以用來製作預算圖表與儀表板，但這類工具通常需要使用者先自行清理資料、設計資料模型與判斷規則。對於學生會預算表這種具有合併儲存格、階層式科目與多列說明的 Excel 格式而言，若沒有先進行資料解析，仍難以直接用於審查。

本專題希望在 Excel 既有資料的基礎上，自動完成資料清理、階層解析、跨年度比較與規則判定，進一步降低審查者整理資料與檢查公式的成本。

### 預期功能

1. **歷年預算與決算資料匯入**
   - 匯入 Excel 格式的預算表與決算表
   - 支援本期預算與上期決算的規則式比較
   - 支援多年度預算表匯入，用於跨年度趨勢比較

2. **預算架構瀏覽**
   - 顯示預算中的階層結構，例如款、項、目、節
   - 支援部門、單位、計畫群、計畫與科目層級的整理
   - 可逐層查看各單位底下的預算明細

3. **科目層級瀏覽**
   - 可從科目角度查看各單位與各計畫的預算分布
   - 例如查看所有「工作費」在哪些單位與計畫底下被編列
   - 協助審查者快速掌握同一科目在不同單位中的差異

4. **搜尋與查詢**
   - 可依科目名稱、單位、計畫、小組或關鍵字搜尋預算項目
   - 快速定位特定支出或收入項目
   - 搜尋結果會顯示金額、說明與審查結果之建議

5. **跨年度比較**
   - 支援上傳多年度預算表
   - 比較各年度中不同審查單位或小組的各科目預算變化
   - 顯示年度差額、成長率與趨勢圖
   - 可用於觀察某單位特定科目是否逐年增加

6. **規則式審查判定**
   - 依照預先設定的審查規則，自動標記可能不合理的預算項目
   - 支援人事相關費用合理性審查
   - 支援低執行率人事科目凍結建議
   - 顯示觸發原因、比較基準與建議凍結金額

7. **凍結建議功能**
   - 對於超出審查規則的部分，自動計算建議凍結金額
   - 將系統產生的建議回填至「審查結果之建議」欄位
   - 提供學代作為審查時的初步參考

8. **優先審查清單**
   - 自動整理所有被規則觸發的預算項目
   - 顯示觸發規則、審查理由與建議凍結金額
   - 協助使用者快速掌握高風險項目

9. **資料結構與演算法效能比較**
   - 支援 Linear Search 與 Hash Map / Inverted Index Search 的搜尋效能比較
   - 顯示不同搜尋方法在同一份預算資料上的查詢時間
   - 作為本專題與資料結構課程內容的連結

### 遇到的困難
目前 prototype 已經能完成基本的資料整理、跨年度比較與規則式審查判定，但在實際設計審查規則時，也遇到一些限制。

首先，現有規則仍然偏向機械化判斷。雖然系統可以透過上期決算、本期預算、非會費收入與執行率等資料計算建議凍結金額，但預算審查本身並不只是數字比較。有些項目可能因新增業務、新增計畫、活動規模擴大或財源結構改變，而使本期預算大幅增加。若系統只依照固定比例或固定門檻判斷，就可能出現「多砍」或「少砍」的問題。

其次，若某一年度行政部門新增大量計畫，整體預算變化可能非常大。在這種情況下，單純用整體預算成長率、人事費成長率或上期決算比例進行判斷，可能無法準確反映預算增加是否合理。例如某部門因新增專案補助或新增活動而增加人事費，系統可能會誤判其為過度膨脹；反之，若某些項目雖然沒有明顯超過數值門檻，但實際上缺乏合理說明，系統也可能沒有標記出來。

第三，學代會在實際審查預算時，不能只是以「科目」為單位統一刪減或凍結。即使系統發現某單位的工作費、臨時工資、稿費或製圖費總額偏高，實際凍結仍然必須回到具體計畫中處理。也就是說，系統可以在「單位 × 科目」層級發現風險，但最後仍需要逐計畫判斷應凍結哪一筆預算。這使得系統產生的建議凍結金額，較適合作為審查輔助與警示，而不能直接等同於正式審議結果。

因此，目前 prototype 的定位仍然是「輔助審查工具」，而不是完全自動化的預算刪減系統。它可以幫助學生代表快速找出值得注意的項目，但仍需要審查者根據預算說明、計畫內容、實際業務需求與政治判斷進行最後決定。

### 下一步計畫

## 下一步計畫

下一步計畫是引入 AI 輔助判斷，使系統不只依靠固定數值規則，而能進一步理解預算說明、計畫內容與過往審查結果。

具體而言，未來可以將 113-2 的預算審議結果作為訓練或參考標準。該次審查是我們第一次較為實質且認真地進行預算審查，因此其審查結果可以作為系統判斷「何種預算應被認為需要說明、刪減或凍結」的重要標的。
由於 113-2 是由我們參與審議，其審核標準較符核合我們之期待，審查標準也相對嚴格，能讓 AI 模型學習較嚴謹的審查尺度，避免系統過度寬鬆。

未來 AI 輔助功能可以朝以下方向發展：

**引入歷史審查結果作為參考**

將 113-2 預算審議結果整理成資料集，對照當時的預算項目、審查意見與最終處理方式，讓系統學習哪些類型的預算較可能被要求說明、刪減或凍結。

**結合規則判定與語意判斷**

現有規則負責快速找出數值異常項目，AI 則進一步閱讀預算說明與計畫內容，判斷該項預算增加是否具有合理原因。

**協助將科目層級風險轉換成計畫層級建議**

目前系統能發現某單位某科目總額偏高，但正式審查時，學代會不能只以科目為單位統一刪減或凍結，而是需要逐計畫審查。因此，未來可讓 AI 根據各計畫說明、金額與歷史審查結果，建議應優先檢視或凍結哪些具體計畫。

**降低多砍與少砍的風險**

對於新增計畫或新增財源，AI 可協助判斷其合理性；對於雖未觸發數值規則但說明不足的項目，AI 也可標記為需要人工審查。如此一來，系統能從單純的機械判斷，逐步變成更接近實際審查流程的輔助工具。

**建立更完整的審查建議輸出**

未來系統不只輸出建議凍結金額，也應輸出「為什麼值得審查」、「可能應要求補充什麼資料」、「是否適合逐計畫凍結」等資訊，讓學生代表能更快形成具體審查意見。

總體而言，下一階段的重點會從「規則式數值判定」進一步擴展到「結合歷史審議結果與 AI 語意判斷的審查輔助」。這樣可以讓系統在保留效率的同時，也更接近實際學代會預算審查的需要。
